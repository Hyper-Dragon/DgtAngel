Virtual Clocks

The virtual clock requires that port 37964 is open. Windows should prompt you to open this port on the firewall, but if not it can be configured manually by opening a command prompt (as an Administrator) and using the appropriate command below.

Open a port

netsh advfirewall firewall add rule name="Dgt Angel ALLOW Tcp Port 37964" dir=in action=allow protocol=TCP localport=37964

Verify if a port is open

netsh advfirewall firewall show rule name="Dgt Angel ALLOW Tcp Port 37964"

Close the port

netsh advfirewall firewall delete rule name="Dgt Angel ALLOW Tcp Port 37964"
